Overview

This documentation covers the Coffee Ordering web API services

RFC2119 Keywords

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in RFC2119.

Media Type Support

All server response bodies and request MUST be valid JSON Media Type messages.

URIs and Operations

Below are the URIs and the operations associated with them.

Orders

The Order (/orders) is base URI for reading and updating Order resources. The table below show all the Order-related operations, their URI, methods, and payloads.
Operation URI Method Input(s) Output(s)
get_orders /orders GET none Order Payload
create_order /orders POST [coffee_id],[additions] Order Payload
get_orders_bystatus /orders/status/{status} GET none Order Payload
get_order_byid /orders/{order_id} GET none Order Payload
update_order /orders/{order_id} PUT [coffee_id],[additions] Order Payload
update_order_status /orders/{order_id} PATCH [status] Order Payload
delete_order /orders/{order_id} DELETE none [order_id]

Payments

The Payments (/Pyments) is base URI for reading and updating Payments resources. The table below show all the Payments-related operations, their URI, methods, and payloads.
Operation URI Method Input(s) Output(s)
create_payment /payments/{order_id} POST [payment_type],[card_name],[card_num],[card_exp] Payment Payload
get_payment /payments/{order_id} GET none Order Payload,Payment Payload
